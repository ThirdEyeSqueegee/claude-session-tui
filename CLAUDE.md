# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

`cst` is a two-pane terminal UI for browsing and resuming Claude Code sessions. It reads every `*.jsonl` transcript under `~/.claude/projects`, presents them grouped by project, and on `↵` `chdir`s into the session's project dir and launches `claude --resume <id>`.

## Commands

```sh
make build   # version-stamped binary (-X main.version from git describe)
make install # build + install to ~/.local/bin
make test    # go test ./...
make race    # go test -race ./...   (CI also adds -shuffle=on)
make vet     # go vet ./...
make fmt     # gofmt -w .
```

Run a single test (one package, all files are `package main`):

```sh
go test -run TestValidID .
```

CI gates every push/PR and all must pass: `gofmt -l` must be empty, `go vet`, `go build`, `go test -race -shuffle=on` (macOS + Linux), `golangci-lint` v2 (config in `.golangci.yml`, top-level `version: "2"`), `govulncheck`, `actionlint`. Always run `make fmt vet race` before a PR.

## Dependencies and gotchas

- TUI built on Bubble Tea v2. Import paths are `charm.land/bubbletea/v2`, `charm.land/bubbles/v2`, `charm.land/lipgloss/v2` — **not** the old `github.com/charmbracelet/*` paths. Don't "fix" these.
- Requires Go 1.26+. `go.mod` is the single source of truth for the version; CI installs exactly that. Module mode is `readonly` in lint — don't expect CI to mutate `go.mod`/`go.sum`.

## Architecture

Single flat `package main`. Data flows: parse transcripts → build model → run Bubble Tea loop → on quit, the chosen session drives the launch.

- **`main.go`** — flag parsing, config load, runs the `tea.Program`, then acts on the result. `cst` does **not** exec-replace itself with `claude`; it runs `claude` as a child and _waits_, so it can tint the kitty tab before and restore after (see kitty.go). `--print`/`--output` let a wrapper own the launch instead. The `prune` subcommand (alias `p`) runs the orphan sweep and exits before the TUI: it lists what it found, then asks `y / N` before removing anything. Exit `130` = quit without choosing.
- **`session.go`** — `parseSession` folds one jsonl into a `Session` (title, project path, msg count, branch, model, first/last message, on-disk `Size`, and summed token `usage`). `loadSessions(gitStatus)` globs and parses all transcripts across a bounded `errgroup` (`SetLimit(min(NumCPU, 8))`, each task writes its own slice index — no shared-state race), off the UI thread, then runs `annotateGit` when `gitStatus` is set. Title precedence: customTitle > aiTitle > first user message. `projectsFingerprint` is a read-only hash over transcript path/size/mtime that both watch paths compare to decide whether a reload is real (gating out chmod/temp-file/no-op events). **JSON on the parse hot path (this file + `renderTranscript` in text.go) uses `github.com/go-json-experiment/json` (json/v2), not stdlib `encoding/json`** — stdlib `encoding/json/v2` is gated behind `GOEXPERIMENT=jsonv2` in Go 1.26 and won't compile under a plain `go build`; this module is the same upstreamed code, importable today. Cold paths (cleanup.go, history.go) stay on stdlib. `contentText` keeps `content` raw (`jsontext.Value`) and only decodes when a byte prefilter (`bytes.Contains(raw, "text")`) proves a text block might exist — tool-only assistant turns (often the largest) skip the array decode entirely. A false positive costs a wasted decode, never a wrong result. Together: ~2.6× fewer allocs per transcript parse.
- **`watch.go`** — live reload of `~/.claude/projects`. Primary path is **fsnotify** (event-driven, near-instant); the time-based poll in ui.go (`watchTickMsg`) is the fallback when a watcher can't be built (inotify limits, unsupported fs). fsnotify is **not recursive** and transcripts sit two levels down (`projects/<encoded-cwd>/<id>.jsonl`), so `startWatchCmd` watches `projects/` itself plus every existing `projects/*` subdir, and `onFSEvent` `Add`s a newly-created project dir on the fly. A burst is debounced (`watchDebounce`) into one `fsReloadMsg`, and every reload is still fingerprint-gated. The watcher is closed in `main.go` before `claude` launches, since cst stays alive as claude's parent for the whole session.
- **`git.go`** — `annotateGit` flags `PathGone` (project dir missing) and `BranchGone` (recorded branch absent from the repo). Deduped per unique project dir: one `os.Stat` and one `git for-each-ref` per repo, not per session. A non-repo dir or missing `git` leaves `BranchGone` false — absence of proof isn't proof of absence.
- **`usage.go`** — token/cost math for the detail pane. Per-1M-token USD pricing by longest model-id prefix (sourced from the `claude-api` skill); cache reads bill ~0.1× input, writes ~1.25×. An unknown model shows token counts without a dollar figure rather than guessing. `humanSize`/`humanCount`/`humanCost` are the display formatters.
- **`config.go`** — TOML config (resolved: `-config` flag → `$CST_CONFIG` → `$XDG_CONFIG_HOME/cst/config.toml` → `~/.config/cst/config.toml`). Merges over defaults; a missing file is fine, a bad value keeps the default and surfaces a warning. Unmarshals into a **zero** Config (not pre-seeded) so an absent pointer-bool stays nil and is distinguishable from an explicit `false` — `mergeDefaults` then fills nils, `boolOr` resolves them at use. Schema mirrored in `config.example.toml`.
- **`ui.go`** — the `model` and Bubble Tea state. The left pane is a flat `[]row` of `rowHeader`/`rowSession`; cursor movement skips headers. `rebuild()` is the single pipeline: filter (AND-substring over a precomputed lowercased `Haystack`) → sort (`recency`/`project`/`msgs`/`size`, cycled by `s`) → group by `groupKey` (`project`/`date`/`branch`, cycled by `S`). Project scope (`.`) filters to sessions whose symlink-resolved cwd matches the launch dir. `loadCmd(gitStatus)` is a closure so a reload keeps the same annotation behavior; `watchCmd` schedules a `watchTickMsg` on the poll interval when `watch` is on.
- **`view.go`** — the `Update`/`View` halves. Four modes: `modeList`, `modeFilter`, `modeTranscript`, `modeConfirmDelete`. `y`/`Y` copy the session id / project path via `tea.SetClipboard` (OSC-52, no dep) with a transient `notice` in the help bar. The transcript pager has in-view search: `/` focuses `searchInput`, `n`/`N` step matches via `jumpToMatch` (case-insensitive, wraps both directions) over `transcriptLines` (an ANSI-stripped copy aligned to the viewport). Live reload is handled by the messages in watch.go (`watchStartedMsg`/`fsEventMsg`/`fsReloadMsg`, with `watchTickMsg` as the poll fallback); each reload is gated on `projectsFingerprint()` moving. Two-pane layout collapses to a single full-width list below `minTwoPaneWidth`. Delete acts on the marked set, or the cursor row when nothing is marked; a row leaves the list only when its on-disk delete actually succeeds.
- **`text.go`** — width-aware (grapheme/ANSI) `truncate`/`wrapClip`/`fitLines`, and `renderTranscript` for the pager.
- **`kitty.go`** — kitty tab coloring via `kitten @ set-tab-color`. Degrades to a no-op outside kitty or when `kitten` is absent. `per_project` derives a stable warm hue from the project path.
- **`cleanup.go`** — `deleteSession` removes a transcript plus its satellite state: `session-env/<id>`, `file-history/<id>`, subagent dirs `projects/*/<id>`, `tasks/session-<id[:8]>`, `jobs/<id[:8]>`, `teams/session-<id[:8]>`, the `sessions/<pid>.json` matched on its inner `sessionId` field (not the pid filename), `paste-cache`/`tasks`/`todos` matches, and the enclosing `projects/<encoded-cwd>` dir when it goes empty. It also strips the session's prompt lines from `history.jsonl` (see history.go). Mirrors the `cce` nushell helper. Note: `tasks`/`jobs`/`teams` and `sessions` are keyed differently from everything else — the first three by the 8-char short id, `sessions` by an id read from inside the file — so don't assume full-UUID-basename keying. All three short-id-keyed removals (`tasks/session-<id[:8]>`, `jobs/<id[:8]>`, `teams/session-<id[:8]>`) are gated by `otherLiveShares`: the short id is only 32 bits, so they're skipped when another live transcript shares the prefix, to avoid reaping a live session's state.
- **`history.go`** — `history.jsonl` is the one satellite that isn't a path per session: all sessions append prompt lines to this single global file, each line carrying a `sessionId`. So cleanup rewrites the file dropping matching lines rather than `RemoveAll`-ing a path. `rewriteHistory(root, keep, apply)` keeps lines whose sessionId satisfies `keep` and returns the drop count; `deleteSession` passes `keep = id != s.ID`, the sweep passes `keep = no-id || live[id]`. A line with no/unparseable sessionId is always kept (can't prove it's an orphan — mirrors the `sessions/*.json` policy). Writes are atomic (temp file + rename, original mode preserved) so a crash can't truncate history, and a scan error aborts the rewrite untouched. A rewrite that drops nothing skips the write entirely.
- **`orphans.go`** — the inverse of `deleteSession`: `findOrphans`/`sweepOrphans` scan the satellite dirs for state whose id has no live transcript (the live set = basenames of `projects/*/*.jsonl`) and reap it. `history.jsonl` is swept apart from the path list (it's a shared file, not a per-id path, and must never be `RemoveAll`'d): `SweepResult.HistoryLines` counts the orphaned prompt lines, `sweepOrphans` strips them via `rewriteHistory` on apply. Drives the `prune` subcommand, which lists orphans (and orphaned history lines) then confirms before removing. All path removals route through `confinedRemover`. Mirrors the `ccp` / `cce-find-orphans` nushell helpers. `shell-snapshots` is deliberately not swept — its filenames carry no session id, and nothing on disk reverse-maps a snapshot to a session. `stats-cache.json` is likewise left alone: it's a rolling daily aggregate that names sessions but has no per-session entry to reap. The short-id-keyed sweeps (`tasks`/`teams/session-<short>`, `jobs/<short>`) match only through `validShortID` (8 hex chars), so non-id siblings the `jobs/` dir also holds — `jobs/.draft-<hex>`, `jobs/pins.json` — are never treated as reapable. The `projects/*` husk check skips non-directory glob matches (e.g. a stray `.DS_Store`) and spares a project dir that holds a live subagent dir (`hasLiveSubagentDir`), since a live session's subagent state can sit under a different encoded cwd than its own transcript.

## Safety invariants — preserve these when editing

- **Every destructive path is derived only from a UUID id.** `validID` (regex in session.go) gates both `parseSession` and `deleteSession`. A jsonl named `...jsonl` has basename `..`; without this check `filepath.Join` could escape and `RemoveAll` a parent. Don't loosen the id check or derive delete paths before validating.
- **`confinedRemover` (cleanup.go) refuses any path not strictly under `~/.claude`**, and refuses root itself. Defense in depth on top of the UUID check. All deletes go through it.
- **`sanitize` (session.go) is the single chokepoint** every jsonl-derived display string flows through — it strips ANSI escapes and C0 control bytes so transcript content can't retitle the terminal, clear the screen, or ring the bell. Route any new jsonl text through `contentText`/`sanitize`, never render raw transcript bytes. Message bodies go through `contentText`; the other display strings (`Branch`, `Model`, and the display `Path`/`Haystack`) are sanitized at the point they're set in `parseSession`. The `contentText` byte prefilter only *skips* the decode of blocks that provably contain no text — every string it does return still flows through `sanitize`, so the fast path can't leak an un-sanitized byte. `PathReal` stays raw — it's the chdir/scope key, never rendered. Its fast path shares the `isStripped` predicate with the strip loop, so they can't drift out of sync and silently leak a control byte the fast path forgot to list.
- **`maxTextLen` caps every kept string** so a pasted multi-MB blob can't stall render/filter.
